JWT

JWT, Json Web Token

  • 一种开放安全的标准，主要用于多个系统之间传递安全可靠的 json 格式的信息以及鉴权，信息经过加密

  • JWT 是一个 token 字符串，由三部分组成

    • Header 头部：指定 token 的类型和一些加密算法
    • payload：存储用户的一些信息等
    • 签名：对前两者联合进行加密，达到验证发送方和请求方身份，保证信息的完整性、可靠性

应用场景

  • 鉴权：一旦用户登录，后续的操作都需要带上 JWT
  • 信息交换：主要用于多个系统之间传递安全可靠的信息

JWT 的流程

  • 客户端使用账号密码请求登录接口
  • 登录成功后，服务器使用签名密钥生成 JWT，然后返回给客户端
  • 客户端再次向服务端请求其他接口时带上 JWT
  • 服务端接收到 JWT 并且会验证 JWT 的有效性，从而返回响应

JWT 和 Session 的区别

  • session 是存储在服务器上的，服务器通过 cookies 中的 sessionID 来获取到当前会话的用户，对于单台服务器这样是可以的，但是如果是多台服务器的话就会涉及到共享 session 的问题，并且随着认证用户的增多，session 也会占用内存
  • 而服务端是不需要存储 JWT 的，JWT 里包含用户的相关信息，服务端拿到 JWT 验证后就可以获得用户信息，也就实现了 session 的功能，但是相比于 session，JWT 是无状态的，不与机器绑定的，只要密钥足够安全，就能保证 JWT 的可靠性

JWT 会有安全问题吗

  • 由于 JWT 是无状态的，token 是有可能被劫持的并且伪造请求，服务器对此是不知道的，可能会出现安全问题。
