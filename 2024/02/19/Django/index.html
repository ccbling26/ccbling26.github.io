Django

简介

  • Django 是用 Python 实现的一种开源 Web 应用框架，其本质是 socket 服务端，采用 MVT（Model-View-Template） 框架模式

运行方式

runserver

  • 使用 Django 自带的 WSGI Server 运行，主要是在测试和开发中使用

    • 线上采用 Nginx（处理静态请求） + uWSGI（处理动态请求）

  • WSGI：Web Server Gateway Interface，一种协议，定义了 Web 服务器和 Python Web 应用程序之间的接口规范

  • runserver 核心内容（详细步骤）

    • 调用将调用 django.core.servers.basehttp.get_internal_wsgi_application 方法中的 get_wsgi_application() 获取 WSGIHandler

      • 通过 get_internal_wsgi_application() 源码可以看出，Django 会先根据 settings 中的 WSGI_APPLICATION 来获取 Handler
      • 在创建 Django 项目时，Django 会默认创建一个 wsgi.py，而 settings 中的WSGI_APPLICATION 配置也会默认指向这个文件

    • 根据 ip_address 和 port 生成一个 WSGIServer 对象，接受用户请求

uWSGI

  • uWSGI：一个实现了 WSGI 协议的 Web 服务器，支持多种协议和功能

  • uwsgi

    • 一种快速且低延迟的协议，用于将 Web 服务器和 Python Web 应用程序连接起来
    • 一种工具，一个实现了 uwsgi 协议的 Web 服务器，支持多线程、异步 I/O、HTTP/1.1 和 WebSockets 等功能。

  • 部署和配置

    • uwsgi.ini

    • 启动

Gunicorn

  • 一个 Python Web 应用程序的 Web 服务器

  • 支持多种 Worker 类型，包括 sync、gevent 和 eventlet，可以根据需要选择不同的 Worker 类型来实现高性能和高并发

  • 通过使用 pre-fork 模型来实现并发处理。在启动时，它会派生多个子进程来处理 Web 请求，每个子进程都是一个单独的 Python 解释器实例。当有新的 Web 请求到来时，Gunicorn 会将请求分配给其中一个子进程进行处理，从而实现并发处理。

  • 部署和配置

    • gunicorn.conf.py

    • 启动

runserver 具体启动流程

  • 命令 python manage.py runserver 的意思是：执行 python 命令，传入的参数为 manage.py 和 runserver，这两个参数会保存到 sys.argv 中，并在 manage.py 中通过调用 Django 的 execute_from_command_line 方法执行内部实现的 runserver 命令

  • 找到 runserver 对应的模块后，会依次调用从父类继承的 run_from_argv $\rightarrow$ execute $\rightarrow$ handle $\rightarrow$ run $\rightarrow$ inner_run，完成参数解析

  • inner_run 函数会调用 get_handler 方法，该方法将调用将调用 django.core.servers.basehttp.get_internal_wsgi_application 方法中的 get_wsgi_application() 获取 WSGIHandler

  • 根据 ip_address 和 port 生成一个 WSGIServer 对象，接受用户请求

序列化（Serialization）

  • 序列化：将属于各种编程语言的数据类型或对象转换为可通过网络传输或可存储到本地磁盘的数据格式（如：XML、JSON或特定格式的字符串）
  • 反序列化：序列化的相反过程
  • 例子：
    • Python 自带json模块的dumps方法
    • Django 自带的serializers类，将QuerySet格式的数据转化为json格式

自定义序列化器

  • DRF 提供了Serializer类和ModelSerializer类，前者需手动指定需要序列化和反序列化的字段，而后者是根据模型生成

使用 Serializer 类

  • 仅可读字段：进入read_only=True参数或者使用ReadOnlyField字段，指客户端是不需要也不能够通过POST或PUT请求提交相关数据进行反序列化

使用 ModelSerializer 类

  • 隐藏字段：不接受用户的输入，默认情况下也不会出现在输出当中，包含于validate_data中。这里author的默认值为request.user

改变序列化器的输出

  • 使用 source
  • 使用 SerializerMethodField
    • 用于显示模型中原本不存在的字段，类似可读字段
    • 可用于将任何类型的数据添加到对象的序列化表示；不能通过反序列化对其直接进行修改
  • 使用 to_representation
    • 直接修改将要输出的序列化数据
  • 使用嵌套序列化器
  • 设置关联模型深度
  • 序列化日期和时间
